## Commands

```bash
# Install dependencies
flutter pub get

# Run the app
flutter run

# Run tests
flutter test

# Run a single test file
flutter test test/path/to/test_file.dart

# Code generation (models, DI, JSON serialization) — run after adding new @injectable or @JsonSerializable annotations
flutter pub run build_runner build --delete-conflicting-outputs

# Watch mode for code generation during development
flutter pub run build_runner watch --delete-conflicting-outputs

# Build for Android
flutter build apk --release

# Analyze code
flutter analyze
```

## Architecture

Clean Architecture with feature-based organization. Each feature is self-contained under `lib/features/` and follows three layers:

- **Domain**: Entities, abstract repositories, use cases (pure Dart, no Flutter/external dependencies)
- **Data**: Models (extend domain entities + JSON serialization), datasource implementations (remote/local), repository implementations
- **Presentation**: BLoC (event/state/bloc files), pages, widgets

### Feature list

`auth`, `auctions`, `admin`, `bidding`, `my_auctions`, `tickets`, `notifications`, `payment`, `profile`, `search`, `scratch_card`

### Shared code

`lib/core/` contains:
- `network/` — Dio-based `ApiClient` with auth interceptor/token refresh, and a WebSocket client for real-time bidding
- `errors/` — `AppException` hierarchy and `Failure` types used with `dartz` Either
- `usecases/` — Base `UseCase<Type, Params>` abstract class
- `constants/`, `utils/`, `widgets/` — App-wide shared resources

### Dependency Injection

`get_it` + `injectable`. Register dependencies in `lib/injection_container.dart`. Classes annotated with `@injectable`, `@lazySingleton`, etc. have code generated by `injectable_generator`. Re-run `build_runner` after adding/changing registrations.

### State Management

Flutter BLoC pattern throughout. All BLoCs use `Equatable` states. Admin-specific BLoCs are injected via `AdminProviders` widget wrapping admin routes — do not register them globally.

### Routing

`go_router` configured in `lib/app/app_router.dart`. Auth-based redirects read from `AuthBloc`. Admin routes are under `/admin/` and use a separate shell from the user-facing bottom nav shell. Feature routes are defined per-feature in `*_routes.dart` files.

### Error Handling Pattern

Repositories return `Either<Failure, T>` (dartz). Data sources throw typed `AppException` subclasses. Use cases propagate Either to BLoCs, which map Left to error states.

### Backend & Real-time

- **Firebase Auth** — authentication
- **Firestore** — auction/bid data with real-time streams
- **Firebase Storage** — images
- **FCM** — push notifications
- **Stripe** (`flutter_stripe`) — payment processing
- **WebSocket** — live bid updates during active auctions

### Internationalization

Supports Dutch (`nl`), English (`en`), and Arabic (`ar`, RTL). Locale is managed by `LocaleBloc`.
